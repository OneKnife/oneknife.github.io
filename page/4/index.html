问题解决

Swift 项目中导入 CommonCrypto

CommonCrypto是一个系统C语言库，在OC项目中可以直接#import<CommonCrypto/CommonCrypto.h>

正常在Swift项目中也可以在桥接文件中引入。可是制作framework没有桥接文件。这就有些麻烦了。

最终解决方案如下：

方法一：

  1. 创建一个Framework 工程

  2. 在framework工程中添加一个新的target, 选择 Cross-platform - Aggregate, 命名为 CommonCryptoModuleMap

  3. 对CommonCryptoModuleMap 添加一个 Run Script

  4. 在MyFramework的 Build Phases - Target Dependencies 中添加CommonCryptoModuleMap 的依赖

  5. 最后在Build Settings -> SearchPaths -> Header Search Paths 中添加 ${BUILT_PRODUCTS_DIR}/CommonCryptoModuleMap 和 $(inherited)

这样在framework中就可以import CommonCrypto了

但是使用这个方法必须要将 三方库编译成.framework包才可以。为了方便在开发中可以reveal代码，我不打算把它打成.framework包，所以这个方法废弃

方法二:

在.podspec 中添加一个脚本。在编译前将CommonCrypto 的头文件写入modulemap，即可实现引入CommonCrypto 的功能

iOS App 图片格式问题闪退

在iOS9.2左右的手机因不规范的图片格式导致闪退。

  1. 找到并解压导出的ipa文件
  2. cd 到解压后的目录 cd Payload/your.app
  3. 运行 xcrun --sdk iphoneos assetutil --info ./Assets.car >/tmp/Assets.json
  4. 打开Assets.json文件，在终端中输入 open /tmp/Assets.json
  5. search "DisplayGamut" : "P3"

替换掉找到的图片即可

Apple Pay证书有效期

Posted on 2018-10-18

iOS证书更新

Apple Pay证书有效期

Apple Pay Payment Processing Certificate 的有效期是 25个月，比较早接入Apple Pay 的开发者，现在应该都该到了要更新证书的时候了。

Apple 将会在到期前 30/15/7 天发送邮件提醒开发者尽快更新证书。

如何更新证书

  1. 登陆http://developer.apple.com/account
  2. 依次选择 Merchant IDs -> 当前使用的Merchant ID -> Edit
  3. 点击“Create Certificate”
  4. 上传银联下载的CSR文件 (如果还保留着银联下载的CSR文件的可以直接上传，如果没有存档可以在银联后台点击“获取CSR”重新下载，请勿点击重置CSR，会影响线上支付！)
  5. 生成新的证书之后，点击新证书“Activite”， 将会自动Revoke旧证书并启用新证书，不会影响线上支付。
